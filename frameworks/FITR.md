# FITR Framework – Competitive Infiltration Strategy

**Purpose:** Enter and win in competitive markets by finding an underserved wedge, demonstrating undeniable value, and expanding from that foothold.

## The Four Phases

| Phase | Focus | Key Question |
|---|---|---|
| **Find** | Identify the wedge | What high-friction workflow does the incumbent handle poorly? |
| **Infiltrate** | Win the beachhead | Can we deliver 10× better on this one thing? |
| **Transform** | Expand the foothold | Now that we're trusted, what adjacent workflows can we own? |
| **Retain** | Build switching costs | What makes us structurally hard to replace? |

## FITR in Agentic Product Strategy

FITR is the GTM motion for AI products entering markets with established incumbents (Epic, Oracle Health, Salesforce, etc.).

**Find**: The incumbent's product is built for the average case. Find the high-value workflow it handles worst — usually the most complex, most regulated, or most time-sensitive.

**Infiltrate**: Deploy an agent that handles that specific workflow dramatically better. Don't try to replace the platform — integrate with it. Be the layer on top that makes it smarter.

**Transform**: Once trust is earned on the wedge workflow, propose expansion. "We handle your inbox triage — want us to handle prior auth too?"

**Retain**: Build data moats (longitudinal workflow data the incumbent doesn't have), workflow integration depth (removing the agent would break the process), and human trust (clinicians who rely on the agent daily).

## FITR + The Trust Ladder

FITR and the PLG trust ladder (Skill 44) are the same motion from different angles:

| FITR Phase | Trust Ladder Stage | What's Happening |
|---|---|---|
| Find | Stage 1 — Observe | Agent watches workflow, surfaces insights |
| Infiltrate | Stage 2 — Draft | Agent drafts outputs for human review |
| Transform | Stage 3 — Supervised Auto | Agent acts on routine cases autonomously |
| Retain | Stage 4 — Full Deployment | Agent is embedded in daily workflow |

## Wedge Selection Criteria

Score potential wedge workflows (1-5 each):
1. **Pain intensity** — How much does this hurt right now?
2. **Incumbent weakness** — How poorly does the current solution handle it?
3. **AI fit** — Is this genuinely automatable with current AI? (See Skill 40)
4. **Expansion potential** — Does winning here open adjacent workflows?
5. **Switching cost** — Once we're embedded here, how hard are we to remove?

Score ≥ 20/25: Strong wedge. Score 15-19: Conditional. Score <15: Find a better wedge.
