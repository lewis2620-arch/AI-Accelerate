# PLG Framework – Product-Led Growth for Agentic Tools

**Purpose:** Drive adoption and expansion of AI products through value demonstration, not sales pressure. Adapted for agentic systems where trust is the primary growth lever.

## The Four Stages

| Stage | Focus | Goal |
|---|---|---|
| **Trigger** | Surface the right moment to introduce the agent | User encounters a high-friction task the agent can handle |
| **Experience** | First value delivery | Agent demonstrates clear, safe value in a low-risk context |
| **Value** | Repeated value + trust building | User sees consistent quality; begins delegating more |
| **Expansion** | Increased autonomy + broader use | User advocates for agent; organization expands deployment |

## PLG in Agentic Systems: The Trust Ladder

Traditional PLG assumes users can self-serve immediately. Agentic PLG requires earning trust first — users don't expand agents they don't trust.

| PLG Stage | Agentic Equivalent | Trust Signal |
|---|---|---|
| Trigger | Agent surfaces insight without being asked | "It noticed something I would have missed" |
| Experience | Agent drafts something worth using | "I sent that with minor edits" |
| Value | Agent handles routine cases autonomously | "I don't think about this category anymore" |
| Expansion | Agent takes on new workflow categories | "Can we use this for X too?" |

## Growth Loops for AI Products

**Quality Flywheel**: Better output → more use → more feedback data → better model → better output

**Trust Flywheel**: Reliable performance → more autonomy granted → more workflow coverage → higher switching cost → stronger retention

**Network Flywheel** (multi-user): One team's adoption creates visible outcomes → adjacent teams request access → org-wide deployment

## PLG Metrics for Agentic Products

- **Activation**: % of users who let the agent complete at least one task end-to-end
- **Retention**: % of activated users still using agent at day 30/60/90
- **Expansion**: Average number of workflow categories delegated per user over time
- **Advocacy**: % of new users who were referred by existing users
- **Override Rate**: % of agent outputs that are manually corrected (declining = trust building)

## Agentic Upgrade

Agents that instrument their own PLG triggers are the next frontier — an agent that detects "this user always manually does X at the end of Y workflow" and proactively offers to handle X is running its own PLG motion.
