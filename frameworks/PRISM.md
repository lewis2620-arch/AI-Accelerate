# PRISM Framework – Responsible Agentic AI Platform Strategy

**Purpose:** Build safe, scalable agentic systems in regulated environments. Every agent in AI Accelerate runs PRISM checks by default.

## The Five Components

| Component | Focus | Key Questions |
|---|---|---|
| **Platform Readiness** | Is the org ready to deploy agents? | Data quality, integration depth, team capability, change management plan |
| **Responsible AI** | Are guardrails in place? | HITL gates, audit trails, bias checks, escalation design, rollback plan |
| **Impact Alignment** | Does it deliver measurable value? | OKRs defined, baseline measured, success criteria agreed before build |
| **System Orchestration** | Is the architecture sound? | RAG design, agent mesh, memory architecture, tool access controls |
| **Market Fit & ROI** | Does it earn adoption and justify cost? | User trust earned, cost-per-task favorable, expansion path defined |

## PRISM Guardrail Checklist

Run this before any agent goes to production:

**Platform Readiness**
- [ ] Data source quality audited and documented
- [ ] Integration points tested with production-equivalent data
- [ ] Clinical/compliance stakeholders signed off on scope

**Responsible AI**
- [ ] HITL gates placed at all Tier 1 and Tier 2 decision points
- [ ] Audit trail logging active — every agent call, every human decision
- [ ] Rollback procedure tested and documented
- [ ] Adversarial testing completed (Skill 14)
- [ ] Bias audit completed for patient-facing systems (Skill 34)

**Impact Alignment**
- [ ] North Star metric defined
- [ ] Baseline measured before deployment
- [ ] Success threshold defined (what does "working" look like at 30/60/90 days?)

**System Orchestration**
- [ ] Agent memory architecture documented (Skill 18)
- [ ] Tool access scoped to minimum necessary (Skill 19)
- [ ] Escalation paths tested end-to-end (Skill 20)

**Market Fit & ROI**
- [ ] Pilot champion identified
- [ ] Trust ladder stage defined (Stage 1-4, see Skill 44)
- [ ] Cost-per-task modeled at scale (Skill 04)

## PRISM Risk Tiers

| Tier | Description | PRISM Requirement |
|---|---|---|
| **Tier 1 — Critical** | Clinical decision support, financial advice, safety systems | Full PRISM checklist + external audit |
| **Tier 2 — High** | Workflow automation with patient/customer impact | Full PRISM checklist |
| **Tier 3 — Medium** | Internal tools, analytics, drafting assistance | Abbreviated checklist |
| **Tier 4 — Low** | Administrative automation, formatting, scheduling | Basic logging + rollback |
