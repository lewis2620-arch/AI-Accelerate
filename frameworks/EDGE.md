# EDGE Framework – Outcome-Driven Product Operations

**Purpose:** Shift product teams from project delivery to measurable business outcomes.

## The Four Phases

| Phase | Focus | Key Activities |
|---|---|---|
| **Empathize** | Deep user insight | Journey mapping, discovery interviews, Jobs-to-be-Done, workflow analysis |
| **Define** | Problem framing | Opportunity sizing, OKR setting, problem statements, opportunity-solution trees |
| **Govern** | Roadmap governance | Intake rituals, prioritization, tradeoff documentation, stakeholder alignment |
| **Evolve** | Experimentation | Hypothesis testing, iteration loops, instrumentation, continuous improvement |

## When to Use Each Phase

**Empathize** — Starting a new initiative, entering a new market, when user behavior is surprising or unclear.

**Define** — Before any build decision. Never skip. This phase prevents expensive wrong turns.

**Govern** — Ongoing. Every sprint, every roadmap update, every stakeholder review.

**Evolve** — Post-launch. Every feature needs an evolution plan before it ships.

## EDGE in Regulated Industries

In healthcare and financial services, each phase has additional requirements:

- **Empathize**: Include compliance and clinical stakeholders in discovery — they surface constraints engineers won't
- **Define**: Problem statement must include regulatory scope (HIPAA, FDA SaMD, OCC) explicitly
- **Govern**: Change management is a first-class workstream — clinical staff don't adopt what they don't trust
- **Evolve**: A/B testing in clinical contexts requires IRB-equivalent review depending on risk tier

## Agentic Upgrade

In agentic product development, EDGE maps to:
- **Empathize** → Agent discovery crew (researcher agents mapping workflows)
- **Define** → Goal vector definition (Skill 21) — outcome-oriented, not feature-oriented
- **Govern** → HITL gate design (Skill 17) + risk classification (Skill 32)
- **Evolve** → Continuous eval pipeline (Skill 31) + drift detection (Skill 30)
