# Problem Statement

**Purpose:** Clearly articulate the customer problem in a way that drives alignment and agentic execution.

**Key Concepts:** Problem vs. solution framing, Jobs-to-be-Done, outcome-oriented language.

**Application:** Use when starting discovery or feeding an agentic workflow.

**Examples:** "Clinicians lose 2 hours daily to inbox triage" (large healthcare network).

**Common Pitfalls:** Jumping to solutions, vague language, ignoring regulatory constraints.

**References:** Marty Cagan, Teresa Torres.

**Product Edge Extension:** Maps to EDGE Empathize phase.  
**Agentic Upgrade:** Feed this directly into multi-agent discovery crew with PRISM guardrails.  
**Regulated Example:** Used in clinical workflow projects to ensure HIPAA-aligned problem scoping.
